單細胞 Foundation 模型

scGPT、Geneformer、scFoundation、UCE——單細胞資料的「ImageNet 時刻」。但記得，傳統 Seurat 還沒過時。

scGPT, Geneformer, scFoundation, UCE — the "ImageNet moment" for single-cell data. Just remember: Seurat is not obsolete.

術語速查

單細胞 Foundation 模型必懂十詞

這 10 個詞貫穿整章——先讀過一次，後面 zero-shot annotation、perturbation prediction、leakage 的討論就會清晰得多。

These ten terms recur throughout. Skim once and the zero-shot annotation / perturbation / leakage discussion below flows easily.

🧫

scGPT

Cui et al., Nature Methods 2024（DOI 10.1038/s41592-024-02201-0）。在 33M cells 上預訓練的 transformer，採 value binning + masked value prediction，在 annotation / integration / perturbation 三項上是當前最常被引用的 scFM。

Cui et al., Nature Methods 2024 (DOI 10.1038/s41592-024-02201-0). Transformer pre-trained on 33M cells with value binning + masked value prediction; the most-cited scFM across annotation, integration, and perturbation.

🧮

Geneformer

Theodoris et al., Nature 2023。把每個細胞表示成「依表達量排序的基因 list」當 token，預訓練於 Genecorpus-30M。GRN / perturbation 預測任務的代表，特別擅長 in-silico TF knockout。

Theodoris et al., Nature 2023. Represents each cell as a rank-ordered gene list (by expression) tokenised on Genecorpus-30M. Flagship for GRN / perturbation prediction; particularly strong on in-silico TF knockout.

🌌

scFoundation

Hao et al., Nature Methods 2024。1.7B 參數、50M cells 預訓練，目前最大的 scFM。設計重點在「read-depth aware」embedding（xTrimoGene encoder），對低 depth 樣本相對 robust。

Hao et al., Nature Methods 2024. 1.7B parameters on 50M cells — the largest scFM to date. Designed as read-depth aware (xTrimoGene encoder), relatively robust to low-depth samples.

🌍

UCE

Universal Cell Embeddings (Rosen et al., bioRxiv 2023 / Cell 2024)。33L transformer，跨 8 物種、36M cells 預訓練；最大賣點是 zero-shot 跨物種對齊，不要求共同基因集。

Universal Cell Embeddings (Rosen et al., bioRxiv 2023 / Cell 2024). 33-layer transformer pre-trained across 8 species and 36M cells; standout feature is zero-shot cross-species alignment without a shared-gene-set requirement.

🪶

CellPLM

Wen et al., ICLR 2024。引入「cell-cell relationship」的 transformer：把鄰近細胞當作上下文（類似 sentence-level transformer 把句子當上下文），在 imputation 與 spatial 任務上有優勢。

Wen et al., ICLR 2024. A transformer that injects cell–cell relationships — using neighbouring cells as context (analogous to sentence-level transformers using sentences as context). Strong on imputation and spatial tasks.

🎯

Zero-shot Annotation

不對 query 資料做任何 fine-tune，直接用預訓練 embedding + reference KNN 給標籤。是 scFM 最直觀的用法，但對「資料 distribution 與 pre-train corpus 差很大」的 disease cells 風險較高。

Label query cells via pre-trained embeddings + reference KNN, with no fine-tuning. The most direct scFM use case, but risky when the query distribution differs substantially from the pre-training corpus (e.g., disease cells).

🧪

Perturbation Prediction

給定基因 KO / 過表達 / drug，預測細胞 state 的 post-perturbation 表現。在 Perturb-seq 資料上 fine-tune；GEARS（Roohani 2024）與 scGPT / Geneformer 是主流 baseline。

Predict post-perturbation cell state from a gene KO / overexpression / drug. Fine-tuned on Perturb-seq data; GEARS (Roohani 2024), scGPT, and Geneformer are the standard baselines.

🕸️

GRN (Gene Regulatory Network)

基因之間的調控關係圖。Geneformer 利用 attention weights 推導 gene-gene relationship；傳統流程用 SCENIC（Aibar 2017）/ pySCENIC，兩者輸出可互補但不該互相替代。

Gene-to-gene regulatory graph. Geneformer derives gene–gene relationships from attention weights; classical pipelines use SCENIC (Aibar 2017) / pySCENIC. Outputs complement rather than replace each other.

📚

Token Vocabulary (Gene)

scFM 的 token 不是「文字」，而是「基因 + 表達量級」的編碼。scGPT 把表達量分 bin 後再做 embedding；Geneformer 純粹用 rank（不存原始值）。兩種設計有不同的歸納偏差。

scFM tokens encode "gene + expression magnitude", not words. scGPT bins expression then embeds; Geneformer uses pure rank (no raw values). The two designs have different inductive biases.

🗂️

Cell Atlas

大規模 reference dataset，如 Human Cell Atlas、CellxGene Census、Tabula Sapiens。scFM 多數預訓練語料來自 atlas；也因此「query 與 pre-train atlas 重疊」是 leakage 的主要來源。

Large-scale reference datasets — Human Cell Atlas, CellxGene Census, Tabula Sapiens. Most scFM pre-training corpora draw from atlases, which is also why "query overlaps with pre-training atlas" is the dominant leakage source.

⚖️

Batch Effect

不同實驗、protocol、平台帶來的非生物變異。傳統流程靠 Harmony / scVI 校正；scFM 的 embedding 號稱 batch-aware，但實際效果在 Patterns 2025 benchmark 上仍與 scVI 有伯仲之爭。

Non-biological variation from experiments, protocols, platforms. Classical pipelines correct with Harmony / scVI; scFM embeddings claim batch-awareness, but Patterns 2025 benchmarks show they still compete neck-and-neck with scVI.

🛡️

Data Leakage

當 evaluation set 已（部分）出現在預訓練 corpus 中，benchmark 分數會被高估。Kedzierska et al. 2024 (bioRxiv) 警告：Tabula Sapiens 等熱門 atlas 在多個 scFM 預訓練中重複出現，導致 zero-shot 表現難以判讀。

Benchmark scores are inflated when the evaluation set partly appears in pre-training. Kedzierska et al. 2024 (bioRxiv) warns that hot atlases like Tabula Sapiens recur across many scFMs' pre-training corpora, making zero-shot scores hard to interpret.

概覽

為什麼要把 LLM 思路套到單細胞？

單細胞 RNA-seq 一個 cell ≈ 兩萬個基因的高維向量，傳統流程：normalisation → HVG → PCA → batch correct → cluster → marker → annotate。每個樣本都要重做。scFM 的概念是：在 33M+ cells 上一次預訓練，學到「細胞的語言」（gene 排序或 expression 模式 ≈ token），任何下游任務只要 zero-shot 或 light fine-tune。

四大代表：scGPT（Cui et al., Nature Methods 2024, DOI 10.1038/s41592-024-02201-0；33M cells）；Geneformer（基因排序作 token，強於 GRN / perturbation）；scFoundation（1.7B 參數最大）；UCE（Universal Cell Embeddings，跨物種 zero-shot）。2025–2026 多個 benchmark（含 Patterns 2025 與 BioLLM review）顯示 scGPT / Geneformer / CellFM 整體最佳，但 leakage 與訓練尺度的爭論未停。

A single-cell RNA-seq cell is a ~20k-gene high-dimensional vector, and the classical pipeline (normalisation → HVG → PCA → batch correction → clustering → markers → annotation) is reinvented per sample. scFM flips this: pre-train once on 33M+ cells to learn "the language of cells" (gene ordering / expression patterns ≈ tokens), then run downstream tasks zero-shot or with light fine-tuning.

Four flagships: scGPT (Cui et al., Nature Methods 2024, DOI 10.1038/s41592-024-02201-0; 33M cells); Geneformer (gene-ranking tokens, strong on GRN / perturbation); scFoundation (1.7B parameters, largest); UCE (Universal Cell Embeddings, cross-species zero-shot). 2025–2026 benchmarks (Patterns 2025, BioLLM review) place scGPT / Geneformer / CellFM at the top, while leakage and pre-training scale debates continue.

💡
核心心法：scFM 不是 Seurat / Scanpy 的替代品，是補充。當任務是 zero-shot annotation、跨資料整合、perturbation 預測時，scFM 加速顯著；當任務是「明確 marker、清楚 cluster」，Seurat / Scanpy 反而更快、更可解釋。 Core mindset: scFMs are complements, not replacements, for Seurat / Scanpy. They shine on zero-shot annotation, cross-dataset integration, and perturbation prediction; for clear markers and distinct clusters, Seurat / Scanpy remain faster and more interpretable.

# scGPT zero-shot cell type annotation
import scgpt, scanpy as sc

adata = sc.read_h5ad("query.h5ad")
ref   = sc.read_h5ad("reference.h5ad")

model = scgpt.load_pretrained("scGPT_human")
emb_q = model.embed_cells(adata)
emb_r = model.embed_cells(ref)

from sklearn.neighbors import KNeighborsClassifier
clf = KNeighborsClassifier(n_neighbors=15).fit(emb_r, ref.obs["cell_type"])
adata.obs["pred"] = clf.predict(emb_q)
sc.pl.umap(adata, color=["pred"])
# Geneformer fine-tune on a perturbation classification task
from geneformer import Classifier

clf = Classifier(classifier="cell",
    cell_state_dict={"state_key":"condition",
                     "states":["ctrl","KO_TP53"]},
    num_classes=2,
    nproc=8)

clf.prepare_data(input_data_file="data.dataset",
                 output_directory="./prep",
                 output_prefix="tp53")
clf.train_classifier(model_directory="geneformer-12L-30M",
                      output_directory="./ft",
                      num_epochs=3)
# UCE: cross-species zero-shot embedding
from uce import eval_single_anndata

emb = eval_single_anndata(
    adata_path="mouse_liver.h5ad",
    species="mouse",
    model_loc="models/33L_8ep_1024t_1280.torch",
    batch_size=25,
    nlayers=33)

真實案例

兩個深入案例：心肌病標靶與 fine-tune 比輸 Seurat

🫀

案例 1：Geneformer 在擴張型心肌病的 in-silico perturbation 找出治療標靶

Theodoris 等人在 Nature 2023 的 Geneformer 原始論文中，用 Genecorpus-30M 預訓練後，將模型應用於 dilated cardiomyopathy (DCM) 患者心肌細胞的單細胞圖譜。他們挑出一群已知與 DCM 相關的轉錄因子，用 in-silico TF perturbation（把 TF token 從輸入序列中移除或往末端 reorder，模擬「該 TF 失去調控力」）測試哪些干擾能讓 DCM 心肌細胞 embedding 重新接近 healthy 心肌。結果發現 NPPA、PLN 等已知靶點之外，TEAD1（Hippo pathway 下游）與 BAG3 在 in-silico 干擾後表現出顯著的 phenotype rescue 訊號；後續的 iPSC 衍生心肌細胞實驗確認 TEAD1 抑制能部分回復 DCM 表型。啟示：scFM 的價值不只在「標註細胞」，更在於透過 attention / embedding geometry 提出「可實驗驗證的 mechanistic hypothesis」。這也是 2024–2026 多家 biotech（Insitro、Cellarity）大量投資 scFM 的核心理由。

Case 1: Geneformer in-silico perturbation surfaces therapeutic targets in dilated cardiomyopathy

In the original Geneformer paper (Theodoris et al., Nature 2023), the authors applied their Genecorpus-30M-pretrained model to single-cell atlases of dilated cardiomyopathy (DCM) cardiomyocytes. They tested a panel of TFs known to be DCM-associated using in-silico TF perturbation (deleting the TF token from input sequences or reordering it toward the tail to simulate "loss of regulatory influence") and measured which perturbations pulled DCM cardiomyocyte embeddings back toward healthy ones. Beyond known targets like NPPA and PLN, the model flagged TEAD1 (downstream of Hippo) and BAG3 with strong phenotype-rescue signatures; subsequent iPSC-derived cardiomyocyte experiments confirmed that TEAD1 inhibition partially reverses the DCM phenotype. The lesson: scFMs aren't just for cell typing — their real value is using attention / embedding geometry to propose experimentally testable mechanistic hypotheses. This is the core reason biotechs like Insitro and Cellarity invested heavily in scFMs across 2024–2026.

📉

案例 2：Theis lab 2024 benchmark——scGPT fine-tune 在小樣本下竟輸給傳統 marker

2024 年 Heumos / Theis 等人在多個 scIB/OpenProblems 風格 benchmark 上做了系統評估，發現一個違反直覺的結果：在 n < 5000 cells、cell type 邊界本來就清楚（如 PBMC 五大 lineage）的情境下，scGPT 即便經過 fine-tune，annotation accuracy 仍輸給 Seurat FindAllMarkers + manual review 的傳統流程，差異約 3–5 百分點。原因有三：(1) 預訓練 corpus 對 PBMC 已飽和，scFM 沒有資訊增益；(2) fine-tune 在小樣本上易 overfit pre-train manifold，產生「自信但偏移」的標籤；(3) 傳統 marker-based 註釋有 immunologist 數十年積累的 priors（如 CD3+ CD4+ → helper T），這種 domain knowledge 在 scFM 的 attention 裡未必能 surface。啟示：scFM 不是萬靈丹，它在「大樣本、稀有細胞、跨資料整合、perturbation」上發光；在「小樣本、明確 marker」上 Seurat / Scanpy 仍是首選。Patterns 2025 BioLLM review 與 2025 年的 Christensen et al. (Bioinformatics) 也獨立報告了類似的結論。

Case 2: Theis lab 2024 benchmark — fine-tuned scGPT loses to classical markers on small data

In 2024, Heumos / Theis and colleagues benchmarked scFMs on scIB/OpenProblems-style suites and reported a counter-intuitive result: when n < 5000 cells and cell-type boundaries are already clean (e.g., the five major PBMC lineages), even fine-tuned scGPT trails the classical Seurat FindAllMarkers + manual review pipeline by ~3–5 percentage points on annotation accuracy. Three reasons: (1) PBMC is saturated in the pre-training corpus — no information gain from the scFM; (2) fine-tuning on small samples easily overfits the pre-train manifold, producing "confident but biased" labels; (3) classical marker-based annotation embeds decades of immunology priors (CD3+ CD4+ → helper T) that scFM attention may not surface. The lesson: scFMs shine on large samples, rare cells, cross-dataset integration, and perturbation; on small samples with clean markers, Seurat / Scanpy remain the better tool. The Patterns 2025 BioLLM review and Christensen et al. 2025 (Bioinformatics) independently reached similar conclusions.

常見誤區

新手用 scFM 最常踩的五個坑

⚠️
誤區 1：把 zero-shot annotation 當 ground truth
為什麼錯：zero-shot 標籤是 reference KNN 的延伸，不是「真實的細胞身份」。對 disease cells、稀有 subtype、developmental intermediates，KNN 在預訓練 embedding 上的最近鄰未必是正確答案。Kedzierska 2024 顯示在「pre-train 沒見過的疾病樣本」上 zero-shot accuracy 可下降 15–25%。
正確做法：把 zero-shot label 視為「待驗證的 prior」，仍要做 marker check、跨方法 cross-validate（如 SingleR、Azimuth、manual review）。對 high-stakes 結論（如 disease-specific cluster），至少要兩種獨立方法一致。
Pitfall 1: Treating zero-shot annotations as ground truth
Why wrong: zero-shot labels are an extension of reference KNN, not "true cell identity". For disease cells, rare subtypes, and developmental intermediates, the nearest neighbour in the pre-trained embedding may be the wrong answer. Kedzierska 2024 shows zero-shot accuracy drops 15–25% on disease samples unseen during pre-training.
Fix: treat zero-shot labels as a prior to be verified — still do marker checks and cross-validate with other methods (SingleR, Azimuth, manual review). For high-stakes conclusions like disease-specific clusters, require agreement from at least two independent methods.
⚠️
誤區 2：忽視訓練資料 leakage（Tabula Sapiens 重複出現）
為什麼錯：scGPT、Geneformer、UCE 等模型都以 CellxGene Census 或 Tabula Sapiens 為核心預訓練語料；若你又拿 Tabula Sapiens 子集當 evaluation set，benchmark 分數會因 leakage 高估。Kedzierska et al. 2024 (bioRxiv) 系統量化發現多個 scFM 「zero-shot」分數中有 30–60% 來自 leakage 增益。
正確做法：選 evaluation dataset 時主動避開 pre-train 已用過的 atlas；或在自家 lab 收集的新資料上做 held-out 評估。論文中要明確 declare 哪些 dataset 在 pre-train 與 eval 都出現過。
Pitfall 2: Ignoring training-data leakage (Tabula Sapiens recurrence)
Why wrong: scGPT, Geneformer, UCE all pre-train on CellxGene Census or Tabula Sapiens. If you then use a Tabula Sapiens subset as your evaluation set, leakage inflates the benchmark. Kedzierska et al. 2024 (bioRxiv) quantified that 30–60% of some scFMs' "zero-shot" gains come from leakage.
Fix: actively avoid atlases already in pre-training when choosing eval sets, or evaluate on freshly collected in-lab data as a held-out test. Papers should explicitly declare which datasets appear in both pre-training and evaluation.
⚠️
誤區 3：直接拿 mouse 預訓練 model 跑 human
為什麼錯：除了 UCE 是明確 cross-species 設計，其他 scFM（scGPT, Geneformer）的 token vocabulary 與 attention 都受 species-specific gene set 影響。跨物種直接套用會出現「homolog 對齊錯誤、表達 magnitude 校正失敗」，下游 annotation 偏差可達 20%+。
正確做法：跨物種任務優先選 UCE；若必須用 scGPT / Geneformer，需要透過 ortholog mapping（如 biomaRt）將 mouse gene 映射到 human equivalent，並重新做 batch correction 與 sanity check。
Pitfall 3: Using a mouse pre-trained model on human directly
Why wrong: only UCE is explicitly cross-species; the others (scGPT, Geneformer) have token vocabularies and attention shaped by species-specific gene sets. Direct cross-species transfer produces "wrong homolog alignment, broken expression-magnitude calibration", with downstream annotation drift >20%.
Fix: prefer UCE for cross-species tasks. If you must use scGPT / Geneformer, map mouse genes to human orthologs (biomaRt), redo batch correction, and run sanity checks before trusting results.
⚠️
誤區 4：相信 perturbation 預測但不做 ablation
為什麼錯：scGPT / Geneformer / GEARS 在 Perturb-seq benchmark 上看起來表現亮眼，但若把 attention 隨機 shuffle、或把預訓練權重 random init，分數有時只下降 5–10%——意味著「模型主要靠 cell-type prior」而非真正學到 perturbation mechanics（Roohani et al. 2024 NMI 提及這個爭議）。
正確做法：發表 perturbation 預測結果前必做 ablation：(a) random shuffle attention、(b) 用 random init backbone fine-tune、(c) 與 simple baseline（線性模型 + DE genes）對比。差異若 < 5%，請保守 claim。
Pitfall 4: Trusting perturbation predictions without ablation
Why wrong: scGPT / Geneformer / GEARS look strong on Perturb-seq benchmarks, but shuffling attention or random-initialising pre-trained weights can drop scores by only 5–10% — implying the model leans on cell-type priors rather than true perturbation mechanics (Roohani et al. 2024 NMI discusses this controversy).
Fix: before publishing perturbation results, run ablations: (a) shuffle attention, (b) fine-tune from random-initialised backbone, (c) compare to a simple baseline (linear model + DE genes). If the gap is <5%, temper your claims.
⚠️
誤區 5：UMAP 好看就以為 embedding 好
為什麼錯：UMAP 是 local-structure preserving 的非線性投影，視覺上的「清楚分群」並不等於 embedding 在量化指標（silhouette score、ARI、ASW、kBET）上真的優秀。Chari et al. 2023 (Nat Comm) 已警告 UMAP/t-SNE 容易 over-cluster；scFM embedding 在 UMAP 上的「美觀」可能只是 hyperparameter 調出來的結果。
正確做法：報告 embedding 品質時同時給 quantitative metrics（scIB 整套：silhouette、ARI、NMI、kBET、graph connectivity），並把 UMAP 視為視覺輔助而非 evidence。
Pitfall 5: Believing a pretty UMAP means a good embedding
Why wrong: UMAP is a local-structure-preserving non-linear projection; visually clean clusters don't mean the embedding is good on quantitative metrics (silhouette, ARI, ASW, kBET). Chari et al. 2023 (Nat Comm) already warned that UMAP/t-SNE over-cluster easily; an scFM's pretty UMAP can be just a hyperparameter artefact.
Fix: report embedding quality with quantitative metrics (the full scIB suite: silhouette, ARI, NMI, kBET, graph connectivity) and treat UMAP as visual aid, not evidence.
資深讀者深入

進階細節

🔬 Gene token vocabulary 設計、masked LM 在單細胞的對應、leakage 警告、cross-species 限制

① Gene token vocabulary 的兩條路。NLP 的 token 是「字 / sub-word」，但細胞沒有自然的 token 概念。兩種主流設計：(a) Geneformer 的 rank-based token——每個 cell 表示成「依表達量遞減排列的基因 list」，token 是基因 ID，位置編碼 = rank；好處是不存原始 magnitude，對 normalization 敏感度低；壞處是失去 magnitude 資訊。(b) scGPT 的 value-binned token——每個 (gene, value_bin) pair 是一個 token，保留 magnitude 但 vocab 變大（典型 ~20k genes × 50 bins ≈ 1M tokens），需要 hash trick 或 factorized embedding。CellPLM 折衷：用 continuous value embedding + gene embedding 相加。

② Masked language modeling 在單細胞的對應。NLP 是 mask 一些 word 預測；scFM 的對應：scGPT 做 masked value prediction（隨機 mask 一部分基因的 expression bin，預測 bin 值）；Geneformer 做 masked gene prediction（mask 部分 rank position 的 gene token）；scFoundation 做 read-depth-aware MAE（同時 mask + 預測 read depth）。三種 objective 對 embedding 幾何的 shape 不同：scGPT embedding 對 expression magnitude 敏感，Geneformer 對 gene composition 敏感。下游任務該選哪個取決於你要保留哪種訊號。

③ Kedzierska 2024 (bioRxiv) 的 leakage 警告——數字版本。該研究系統檢查 scGPT、Geneformer、UCE 的預訓練 metadata，發現：(a) Tabula Sapiens 出現在 3/3 model 的 pre-training（直接 leakage 風險）；(b) 在 Tabula Muris 的 hold-out split 上，scGPT zero-shot Macro F1 報告 0.78，但去除已見過的 donor / tissue overlap 後降到 0.62；(c) 某些 disease atlas（COVID、cancer）也以混合方式進入 pre-training，使「疾病樣本 zero-shot」評估不可信。建議：論文 reporting 應列出 pre-training corpus 完整 dataset ID + 在 evaluation 中做明確 leakage check（如 cell-id intersection）。

④ Cross-species transferability。UCE 透過 "protein embedding + cross-species attention" 學跨物種 invariant 表示，但這只在「ortholog 對齊得好 + 演化保守通路」時 work。Rosen et al. 自己也承認在 mouse vs zebrafish 這類更遠的物種對上效能下降。實務建議：(a) 跨物種距離越遠，越依賴 manual ortholog mapping；(b) developmental stage matters——同物種不同 stage 的 transfer 也未必 trivial；(c) 對 evolutionary novel cell types（如某些 immune subsets），cross-species transfer 可能完全失敗。

① Two paths for gene token vocabulary. NLP tokens are words / sub-words, but cells have no natural token. Two mainstream designs: (a) Geneformer's rank-based tokens — each cell as a "gene list sorted by descending expression", tokens = gene IDs, position embedding = rank. Pro: no raw magnitude stored, robust to normalization. Con: discards magnitude. (b) scGPT's value-binned tokens — each (gene, value_bin) pair is a token, preserving magnitude but blowing up vocab (~20k genes × 50 bins ≈ 1M tokens), requiring hash tricks or factorised embeddings. CellPLM compromises with continuous value embeddings added to gene embeddings.

② Masked language modeling in single cell. NLP masks words to predict; scFMs' equivalents: scGPT does masked value prediction (mask expression bins, predict them); Geneformer does masked gene prediction (mask gene tokens at rank positions); scFoundation does read-depth-aware MAE (mask + predict read depth). The three objectives sculpt embeddings differently — scGPT is magnitude-sensitive, Geneformer is composition-sensitive. The right choice depends on which signal you need to preserve downstream.

③ Kedzierska 2024 (bioRxiv) leakage warnings — by the numbers. The study audited pre-training metadata for scGPT, Geneformer, UCE: (a) Tabula Sapiens appears in 3/3 models' pre-training (direct leakage risk); (b) on a Tabula Muris hold-out split, scGPT zero-shot Macro F1 reported as 0.78 drops to 0.62 once seen donor/tissue overlaps are removed; (c) some disease atlases (COVID, cancer) also leaked into pre-training, undermining "disease-sample zero-shot" claims. Recommendation: papers should list full pre-training dataset IDs and run explicit leakage checks (e.g., cell-id intersections) in evaluation.

④ Cross-species transferability. UCE uses "protein embeddings + cross-species attention" to learn species-invariant representations, but this only works when orthologs align well and pathways are evolutionarily conserved. Rosen et al. concede performance drops on more distant pairs like mouse vs zebrafish. Practical advice: (a) the farther the species, the more you need manual ortholog mapping; (b) developmental stage matters — same-species cross-stage transfer is not trivial either; (c) for evolutionarily novel cell types (e.g., certain immune subsets), cross-species transfer can fail outright.

Q1: scFM 真的能取代 Seurat 的可解釋性嗎？

scFM 的 embedding 是高維、黑箱的；Seurat / Scanpy 的 marker-based 註釋雖然 manual，但每一步都可解釋（CD3+ → T cell）。某 PI 主張未來 lab 直接學 scGPT，不再教 marker-based 流程；另一 PI 反對：「沒有 marker priors，學生看不懂 paper、也說不出生物意義」。請思考：在訓練學生時，可解釋性與效率該如何平衡？你會讓博士生先學哪個？scFM 與 marker-based 是替代還是互補？

Q1: Do scFMs really replace Seurat's interpretability?

scFM embeddings are high-dimensional and opaque; Seurat / Scanpy marker-based annotation is manual but interpretable at every step (CD3+ → T cell). One PI argues the future lab should learn scGPT directly and skip markers; another counters: "without marker priors students can't read papers or articulate biology." Consider: how do you balance interpretability and efficiency when training students? Which do PhDs learn first? Are scFMs a replacement or a complement to markers?

One reviewer argues every scFM paper must attach a leakage audit table listing pre-training-vs-evaluation intersections; an author pushes back: "too strict, and corpus lists are trade secrets." Q3: Is 1.7B-param scFoundation really better than 33M-param scGPT?

scFoundation claims the largest scFM, but 2025 benchmarks place it within 5 percentage points of scGPT / Geneformer on most tasks. Geneformer DCM 案例顯示 in-silico TF KO 能提示治療標靶，但後續仍需 iPSC 實驗驗證。某計算生物學家認為「in-silico 結果該獨立發表，加速 hypothesis 流通」；某實驗生物學家反對：「未驗證的 in-silico claim 會污染文獻、誤導後續研究」。請思考：in-silico-only 的 scFM perturbation 結果在 paper 中該怎麼定位？該不該強制 wet-lab validation？對 dry lab 與 wet lab 合作的 fair credit 又該如何安排？

Q4: Are scFM in-silico perturbations hypotheses or evidence?
