生物影像 AI

Cellpose-SAM、StarDist、SAM 2、CellSAM——讓細胞分割從「博士論文題目」變成「兩行 code」。

Cellpose-SAM, StarDist, SAM 2, CellSAM — turning cell segmentation from a PhD project into two lines of code.

術語速查

生物影像 AI 必懂十詞

這 10 個詞貫穿整章——先讀過一次，後面 Cellpose-SAM、StarDist、SAM 2、IoU、Dice 的討論就會清晰許多。

These ten terms recur throughout. Skim once and the discussion of Cellpose-SAM, StarDist, SAM 2, IoU, and Dice flows easily.

🧫

Cellpose

Stringer et al. 2021 Nat Methods。CNN-based generalist 細胞分割，核心是「流場」（gradient flow）：模型預測每個 pixel 指向細胞中心的向量，再追蹤匯聚點得到 instance mask。2025 升級為 Cellpose-SAM（SAM ViT backbone）。

Stringer et al. 2021 Nat Methods. CNN-based generalist cell segmentation built on "flow fields" (gradient flow): the model predicts a vector at every pixel pointing to its cell centre, then tracks convergence points into instance masks. Upgraded to Cellpose-SAM (SAM ViT backbone) in 2025.

⭐

StarDist

Schmidt et al. 2018 (MICCAI)。把每個 nuclei 表示為「中心點 + N 條 radial distance」的 star-convex polygon，對 H&E、IF nuclei 等 convex shape 效能極佳。2D_versatile_he 是 H&E 業界標準預訓練。

Schmidt et al. 2018 (MICCAI). Represents each nucleus as "centre + N radial distances" = star-convex polygon, excellent on H&E and IF nuclei (convex shapes). The 2D_versatile_he pretrained model is the H&E industry standard.

🎯

SAM (Segment Anything)

Kirillov et al. 2023 ICCV (Meta)。在 SA-1B（11M images / 1B masks）上預訓練的 ViT-based promptable segmentation foundation model；接受 point / box / mask prompt，輸出任意物體 mask。是 Cellpose-SAM、CellSAM 的 backbone 基礎。

Kirillov et al. 2023 ICCV (Meta). ViT-based promptable segmentation foundation model pretrained on SA-1B (11M images / 1B masks). Accepts point / box / mask prompts; outputs masks for any object. Backbone for Cellpose-SAM and CellSAM.

🎥

SAM 2

Ravi et al. 2024 (Meta)。SAM 的 video 升級版，引入 memory attention 在幀間傳播 mask；對 time-lapse / lightsheet movie 的細胞追蹤，提供「click frame 0 → 全 video 自動 propagate」能力。

Ravi et al. 2024 (Meta). The video upgrade of SAM, adding memory attention to propagate masks across frames. For time-lapse / lightsheet movies, enables "click frame 0 → auto-propagate through the whole video".

🧪

CellSAM

Israel et al. 2025 Nat Methods。CellFinder 偵測器 + SAM prompt 流程：先用 detector 找 cell bounding box，再用 box 當 prompt 餵 SAM。對「擁擠 / overlapping cells」效果好。

Israel et al. 2025 Nat Methods. CellFinder detector + SAM prompting: detector locates cell bounding boxes, then boxes become prompts to SAM. Strong on dense / overlapping cells.

🔢

Instance Segmentation

每個物件得到獨立的 ID（cell 1, cell 2, …）。是計數、tracking、shape analysis 的基礎。Cellpose、StarDist、SAM 都做 instance segmentation。

Each object gets a unique ID (cell 1, cell 2, ...). The foundation for counting, tracking, shape analysis. Cellpose, StarDist, SAM all do instance segmentation.

🎨

Semantic Segmentation

只區分 class（e.g., cell vs background）而不分 instance。對「cell density map」或「tissue type 區劃」夠用，但無法數細胞。U-Net（Ronneberger 2015）是經典代表。

Distinguishes classes (cell vs background) but not instances. Sufficient for cell-density maps or tissue-type zoning, but you can't count cells. U-Net (Ronneberger 2015) is the classical representative.

📊

IoU (Intersection over Union)

分割評估指標：IoU = |A ∩ B| / |A ∪ B|。0 = 完全不重疊、1 = 完全一致。COCO / 細胞分割 benchmark 常用 IoU@0.5 / 0.75。

Segmentation metric: IoU = |A ∩ B| / |A ∪ B|. 0 = no overlap, 1 = identical. COCO / cell-segmentation benchmarks commonly use IoU@0.5 / 0.75.

🎲

Dice Coefficient

Dice = 2|A ∩ B| / (|A| + |B|)。等價於 F1-score 對於分割問題；對小物體較 forgiving。醫療影像（lesion seg）多用 Dice，細胞 benchmark 多用 IoU。

Dice = 2|A ∩ B| / (|A| + |B|). Equivalent to F1 for segmentation; more forgiving on small objects. Medical imaging (lesion seg) leans Dice; cell benchmarks lean IoU.

⛰️

Watershed

1991 年的傳統演算法：把灰階影像當地形圖，從「local minima」灌水擴散，分水嶺線即為物件邊界。Deep learning 出現前的主流 instance separation 方法；至今仍用於 post-processing。

The 1991 classical algorithm: treat the grayscale image as a topographic map, flood from local minima, and the watershed lines are object boundaries. The dominant instance-separation method before deep learning; still used as post-processing.

〰️

Active Contour

Kass et al. 1988 (snakes)。「彈性曲線」演算法：能量函數 = 內部彈性 + 外部影像梯度 + 終止條件，迭代收斂到物體邊界。傳統年代的細胞 / lesion 分割主力，現多被 CNN 取代但仍用於 fine-tuning seed mask。

Kass et al. 1988 (snakes). The "elastic curve" algorithm: energy = internal elasticity + external image gradient + termination, iteratively converging to boundaries. The classical workhorse for cell / lesion segmentation; largely replaced by CNNs but still used to refine seed masks.

🏷️

Annotation

由人類專家在影像上手工畫出 ground-truth mask / box / point。是 supervised training 的成本核心；典型 cell 影像專家標註速度 30–60 cells/hr，dense field 可降到 5–10。Foundation model 的目標就是讓 annotation cost 降到 minimal。

Human experts manually drawing ground-truth masks / boxes / points on images. The cost centre of supervised training; typical experts annotate 30–60 cells/hr on cell images, dropping to 5–10 in dense fields. Foundation models aim to drive annotation cost to minimal.

概覽

為什麼 2025–2026 是生物影像 AI 的拐點？

2020 年的 Cellpose 已經是一場革命——你不再需要手動調 watershed 參數。但 2025 年發表的 Cellpose-SAM (bioRxiv 2025.04.28) 把 SAM transformer 當 backbone 替換掉原本的 CNN，結果是：在多個 benchmark 上「效能超越人類間一致性 (human inter-rater agreement)」——換句話說，AI 與人類專家的差異，已經小於兩個人類專家之間的差異。

同期，CellSAM (Nat Methods 2025) 把 prompt-based SAM 和自家 CellFinder 偵測器結合；StarDist 仍是 H&E nuclei 分割的金標準；Meta SAM 2 把 video / 多幀分割變得近乎免費；BioImage Model Zoo 讓所有模型 ilastik、QuPath、Napari、CellProfiler、deepImageJ 之間互通——選工具的瓶頸從「演算法」變成「modality 對不對 + GPU 夠不夠」。

The 2020 Cellpose was already a revolution — you no longer hand-tune watershed parameters. But Cellpose-SAM (bioRxiv 2025.04.28) swaps the CNN backbone for SAM transformers, and the result is that on several benchmarks it "exceeds human inter-rater agreement" — the model is more consistent with human consensus than two human experts are with each other.

In parallel, CellSAM (Nat Methods 2025) fuses prompt-driven SAM with a CellFinder detector; StarDist remains the gold standard for H&E nuclei; Meta SAM 2 makes video / multi-frame segmentation near-free; the BioImage Model Zoo makes everything interoperate across ilastik, QuPath, Napari, CellProfiler, deepImageJ. The bottleneck has shifted from algorithm to "is the modality right? Is the GPU big enough?"

💡
核心心法：2026 年生物影像 AI 的選型原則：先確定 modality（brightfield / fluorescence / H&E / video） 與 target（whole cell / nuclei / organelle），再從 BioImage Model Zoo 找預訓練模型；幾乎不需要從頭訓。 Core mindset: The 2026 selection rule for bioimage AI: first nail down modality (brightfield / fluorescence / H&E / video) and target (whole cell / nuclei / organelle), then pull a pre-trained model from the BioImage Model Zoo. Training from scratch is almost never needed.

實作程式碼

三、三種典型 pipeline

# Cellpose-SAM — generalist segmentation
# pip install cellpose-sam
from cellpose_sam import models
import tifffile, numpy as np

img = tifffile.imread("colony_brightfield.tif")   # H,W or H,W,C
mdl = models.CellposeSAM(gpu=True)

masks, flows, styles = mdl.eval(
    img, diameter=None,           # auto-estimate
    flow_threshold=0.4,
    cellprob_threshold=0.0)

print(f"Found {masks.max()} cells")
# masks: int array, 0 = background, 1..N = cell IDs
# StarDist 2D — H&E or IF nuclei
# pip install stardist
from stardist.models import StarDist2D
from csbdeep.utils import normalize
import tifffile

mdl = StarDist2D.from_pretrained("2D_versatile_he")

img = tifffile.imread("tumor_HE.tif")
img_n = normalize(img, 1, 99.8, axis=(0,1,2))

labels, details = mdl.predict_instances(
    img_n, prob_thresh=0.5, nms_thresh=0.4)

print(f"Nuclei: {labels.max()}")
# details['coord'] = polygon vertices per nucleus
# SAM 2 — track cells across video frames
# pip install sam2
from sam2.build_sam import build_sam2_video_predictor
import torch

predictor = build_sam2_video_predictor(
    config_file="sam2_hiera_l.yaml",
    ckpt_path="sam2_hiera_large.pt")

state = predictor.init_state(video_path="timelapse/")

# Click one cell on frame 0; SAM 2 propagates the mask
predictor.add_new_points(state, frame_idx=0, obj_id=1,
    points=[[128, 128]], labels=[1])

for idx, ids, masks in predictor.propagate_in_video(state):
    save_mask(idx, masks)        # full track through video

真實案例

兩個深入案例：Cellpose-SAM 從 0.78 拉到 0.91、SAM 2 追蹤 200 cells × 100 frames

🧫

案例 1：Cellpose-SAM 用 SAM ViT-H backbone 把 fluorescence cell segmentation IoU 從 0.78 拉到 0.91

2025 年 Carsen Stringer 團隊在 bioRxiv (2025.04.28.651001) 公開 Cellpose-SAM：他們把原版 Cellpose 的 U-Net backbone 換成 Meta SAM 的 ViT-H（630M 參數，預訓於 SA-1B 1B masks），其他流程（flow field 預測、cellprob 邊界、watershed-like post-processing）保留。在 LIVECell 與 TissueNet benchmark 上，fluorescence cell segmentation 的 IoU 從原版 Cellpose 2.0 的 0.78 提升到 0.91；對於「人類專家 vs 人類專家」的 inter-rater IoU 約 0.86，意味著 Cellpose-SAM 已比兩位人類專家彼此一致還準。技術上關鍵有三：(1) SAM ViT-H 在 SA-1B 上學到的 generic object boundary prior 比 ImageNet pretrained U-Net 強得多；(2) ViT 的 global attention 對「擁擠細胞 / 大尺寸變化」效果更好；(3) Cellpose 的 flow-field decoder 對 SAM feature 仍 transfer 得很順。啟示：generic foundation backbone（SAM、DINOv2）+ domain-specific decoder 是 2025–2026 生物影像 AI 的標準範式；從零訓 backbone 在多數情境已不划算。

Case 1: Cellpose-SAM with a SAM ViT-H backbone lifts fluorescence cell IoU from 0.78 to 0.91

In 2025 the Carsen Stringer team published Cellpose-SAM on bioRxiv (2025.04.28.651001): they swapped Cellpose's U-Net backbone for Meta SAM's ViT-H (630M params, pretrained on SA-1B / 1B masks) while keeping the rest of the pipeline (flow-field prediction, cellprob boundary, watershed-like post-processing). On LIVECell and TissueNet benchmarks, fluorescence cell segmentation IoU jumped from Cellpose 2.0's 0.78 to 0.91; human-vs-human inter-rater IoU is ~0.86, meaning Cellpose-SAM is more consistent with consensus than two human experts are with each other. Three technical keys: (1) SAM ViT-H's generic object-boundary prior from SA-1B beats ImageNet-pretrained U-Net by a wide margin; (2) ViT's global attention handles dense fields and large size variation better; (3) Cellpose's flow-field decoder transfers smoothly from SAM features. The lesson: "generic foundation backbone (SAM, DINOv2) + domain-specific decoder" is the standard paradigm for 2025–2026 bioimage AI; training a backbone from scratch is usually no longer worth it.

🎥

案例 2：SAM 2 在 lightsheet developmental movie 自動追蹤 200 cells × 100 frames

2025 年某 zebrafish 發育生物學 lab 想分析 lightsheet 顯微鏡拍下的 100 幀 time-lapse（4D：x, y, z, t），追蹤胚胎發育過程中 ~200 個神經元前體細胞的 lineage。傳統流程：用 TrackMate 或自製 Hungarian tracker，每個 cell 需 1–2 hr 人工修正；200 cells × 100 frames 至少 100 工時。他們改用 SAM 2 video predictor：只在 frame 0 click 200 個 cell 中心點（約 15 分鐘），模型用 memory attention 把 mask 自動 propagate 到所有後續 frames，輸出每個 cell 的時序 mask 序列。比對人工 ground truth：自動 track 在 95 / 100 frames 上 IoU > 0.7（細胞發生分裂或被擋住時偶爾掉鏈），總共節省約 80% 人工時間。不足之處：(a) 細胞分裂時 SAM 2 不會自動建立新 lineage（仍需 manual split annotation）；(b) 嚴重 occlusion 時 mask drift；(c) GPU memory ~16 GB 才跑得順。啟示：SAM 2 把「video 細胞追蹤」從 PhD-level 課題降到 undergraduate 可勝任；但 cell division、occlusion 等生物特定情境仍需人工介入或專用 tracker（如 TrackPy、btrack）配合。

Case 2: SAM 2 auto-tracks 200 cells across 100 frames in a lightsheet developmental movie

In 2025 a zebrafish developmental biology lab wanted to analyse a lightsheet movie of 100 frames (4D: x, y, z, t) and track ~200 neural progenitor lineages through embryonic development. Classical pipeline: TrackMate or a custom Hungarian tracker, with 1–2 hr of manual correction per cell — 200 × 100 = ~100 person-hours. They switched to the SAM 2 video predictor: just click 200 cell centres on frame 0 (~15 min), and memory attention propagates masks through every subsequent frame, producing per-cell mask time series. Against manual ground truth: automatic tracks hit IoU >0.7 on 95/100 frames (occasional drops on divisions or occlusions), saving ~80% of person-hours. Caveats: (a) SAM 2 doesn't auto-spawn new lineages on division (manual split annotation still needed); (b) severe occlusion causes mask drift; (c) ~16 GB GPU memory required. The lesson: SAM 2 turned "video cell tracking" from a PhD-level project into something an undergraduate can tackle; but biology-specific phenomena like division and occlusion still need human intervention or a dedicated tracker (TrackPy, btrack) in the loop.

常見誤區

新手做生物影像 AI 最常踩的五個坑

⚠️
誤區 1：在 IHC 直接用 H&E pretrain 的 model
為什麼錯：H&E 染色色彩（紫紅 + 粉紅）與 IHC（棕色 DAB + 藍色 hematoxylin）光譜完全不同。StarDist 2D_versatile_he 在 H&E 上 SOTA，但拿到 IHC 上 IoU 通常從 0.85 降到 0.5–0.65，因為 IHC 的 chromatic distribution 是 OOD（out-of-distribution）。
正確做法：(a) 找 IHC 專用 pretrained model（QuPath、HoVer-Net、CellViT）；(b) 若沒有，retrain / fine-tune Cellpose 或 StarDist 在 50–200 張 IHC 標註上，通常一張 24GB GPU 一晚可完成。事前用 ColorConvolution / Macenko stain deconvolution 也可緩解色差。
Pitfall 1: Using an H&E-pretrained model on IHC directly
Why wrong: H&E (magenta + pink) and IHC (brown DAB + blue hematoxylin) have entirely different chromatic profiles. StarDist 2D_versatile_he is SOTA on H&E but typically drops from IoU 0.85 to 0.5–0.65 on IHC because the chromatic distribution is OOD.
Fix: (a) use an IHC-specific pretrained model (QuPath, HoVer-Net, CellViT); (b) otherwise retrain / fine-tune Cellpose or StarDist on 50–200 IHC annotations — typically one night on a 24GB GPU. Pre-processing with Color Convolution / Macenko stain deconvolution also helps.
⚠️
誤區 2：3D 影像沒做 Z-stack 校正就分割
為什麼錯：confocal / lightsheet 的 Z-stack 通常有 anisotropic voxel（XY 0.2 µm、Z 1 µm）、illumination drop along Z（深層訊號弱）、refractive index mismatch 造成的軸向偏移。直接用 2D model 跑每層 → 用 watershed stitch 時，會出現 ghost cells、層間錯位、過度合併等問題。
正確做法：(a) 先做 z-attenuation correction（如 BaSiC、CIDRE）；(b) 用 Cellpose 3D 模式或 StarDist 3D（直接 3D tensor 推論）；(c) 對 lightsheet 多視角資料，先 fusion 再分割（BigStitcher → Cellpose）。
Pitfall 2: Segmenting 3D images without Z-stack correction
Why wrong: confocal / lightsheet Z-stacks have anisotropic voxels (XY 0.2 µm, Z 1 µm), illumination drop along Z (weak signal deep in), and refractive-index-mismatch axial shifts. Running a 2D model per slice and stitching with watershed produces ghost cells, inter-slice misregistration, and over-merging.
Fix: (a) z-attenuation correction (BaSiC, CIDRE) first; (b) use Cellpose 3D or StarDist 3D on the full 3D tensor; (c) for multi-view lightsheet, fuse before segmenting (BigStitcher → Cellpose).
⚠️
誤區 3：把 instance segmentation 當 semantic 用
為什麼錯：U-Net 等 semantic segmentation 模型輸出 binary mask（cell vs background），不分 cell ID。新手常對 binary mask 直接 connected_components 標 instance，但密集細胞時相鄰 cell 被連成一塊，導致細胞數嚴重低估（典型 -20% to -50%）。
正確做法：需要 cell counting / per-cell feature 時，必選 instance segmentation 模型（Cellpose、StarDist、CellSAM）。若手上只有 semantic mask，需先做 distance transform + watershed 或 marker-controlled watershed 才能分離 instance。
Pitfall 3: Treating instance segmentation as semantic
Why wrong: U-Net-style semantic models output a binary mask (cell vs background) with no cell IDs. Beginners often connected_components on a binary mask, but in dense fields adjacent cells get merged — undercounting by 20–50%.
Fix: when you need cell counts or per-cell features, use an instance segmentation model (Cellpose, StarDist, CellSAM). If you only have a semantic mask, separate instances first with distance transform + watershed or marker-controlled watershed.
⚠️
誤區 4：不調整 SAM 的 prompt point 就相信結果
為什麼錯：SAM 的輸出對 prompt 極敏感。同一張影像，prompt point 在 cell 中心 → 完美 mask；prompt 偏到 cell 邊緣 → 模型可能只圈一半、或圈到鄰居；prompt 點落在背景 → 模型可能輸出整片背景 mask。SAM 2 在 video 上的 mask drift 也常源於初始 prompt 不夠精確。
正確做法：(a) 用 detector（CellFinder、StarDist）先給高品質 box / point prompt（CellSAM 路徑）；(b) 對手動 prompt，prompt point 越靠近 cell centroid 越穩；(c) 使用 SAM 的 multi-mask output（每個 prompt 輸出 3 個 mask 候選）+ confidence score 選最好的；(d) 對重要結果，always overlay 視覺檢查。
Pitfall 4: Trusting SAM without tuning prompt points
Why wrong: SAM is highly prompt-sensitive. On the same image, a prompt at the cell centre → perfect mask; near the edge → only half-cell or includes neighbour; on background → whole-background mask. SAM 2 video drift often traces to imprecise initial prompts.
Fix: (a) feed high-quality box / point prompts from a detector (CellFinder, StarDist) — the CellSAM route; (b) for manual prompts, place the point near the centroid; (c) use SAM's multi-mask output (3 mask candidates per prompt) + confidence score to pick the best; (d) always overlay visually on important results.
⚠️
誤區 5：GPU memory 不足卻硬跑全圖 → 邊界 artifact
為什麼錯：WSI（whole-slide image）動輒 100,000 × 100,000 pixel，10–20 GB。新手對 8 GB GPU 想一次處理全圖，被迫 downsample 4–8 倍——導致小 nuclei 完全消失。或者強行 tile 但不設 overlap，tile 邊緣的細胞被切兩半，post-process 又無法合併，最終影像邊界出現假 fragments。
正確做法：用 tile-based inference 並設 10–20% overlap，post-process 用 NMS（non-max suppression）或 mask merging 重組跨 tile 邊界的物件。QuPath、HoVer-Net 都內建這種 tile-stitching 流程。對 WSI，推薦 Cellpose 的 resize 模式或 StarDist 的 block-wise predict。
Pitfall 5: Forcing full-image inference on insufficient GPU → boundary artefacts
Why wrong: WSIs (whole-slide images) are 100,000 × 100,000 px, 10–20 GB. Beginners on 8 GB GPUs often downsample 4–8× to fit — small nuclei disappear entirely. Or they tile without overlap, cells at tile borders get cut in half, post-processing can't merge them, and false fragments accumulate at boundaries.
Fix: tile-based inference with 10–20% overlap, then post-process with NMS or mask merging to reassemble cross-tile objects. QuPath and HoVer-Net include tile-stitching pipelines. For WSI, use Cellpose's resize mode or StarDist's block-wise predict.

🔬 SAM ViT 架構、Cellpose 流場演算法、ImageJ + Python pipeline 整合、JUMP-Cell Painting benchmark

① SAM ViT 架構（Kirillov 2023）。SAM 由三部分組成：(a) Image Encoder——MAE pretrained ViT-H/L/B（632M / 308M / 86M 參數），輸入 1024×1024 image → 64×64×256 image embedding；計算量大但每張圖只跑一次（可 cache）。(b) Prompt Encoder——把 point / box / mask 編成 256-d embedding，sparse（point/box）走 positional encoding，dense（mask）走 conv embedding。(c) Mask Decoder——two-way attention transformer (~4M 參數)，融合 image + prompt embedding，輸出 3 個 mask + IoU prediction，輕量、可即時互動。這種設計讓你可以「encode 一次、prompt 無限次」，是 SAM 在互動式工具（Napari、ImageJ）能用的關鍵。

② Cellpose 流場演算法（Stringer 2021）。傳統 instance segmentation 用 anchor / box；Cellpose 改用「flow field」：對每個 pixel 預測一個 2D 向量，指向該細胞中心。訓練時 ground truth flow 由 ground-truth mask 經 heat diffusion 計算（向 mask center 流動）。Inference 時：(a) 預測 flow + cellprob；(b) 對每個 pixel 跑「Euler integration」沿著 flow 走 200 步；(c) 終點匯聚到 mask centers 形成 cluster——這天然產生 instance ID。優勢：對任意形狀 / 拓撲（細長、彎曲、有孔）都 work，不像 StarDist 限於 star-convex。Cellpose-SAM 保留這個 decoder，只把 backbone 換成 SAM ViT。

③ ImageJ macro + Python pipeline 整合。實務上常用組合：(a) Fiji + Cellpose plugin——Fiji 的 BIOP Cellpose wrapper 可在 macro 中呼叫 Python conda env 的 Cellpose，輸入 ROI 輸出 mask；(b) QuPath + StarDist extension——QuPath 的 Groovy script 可直接呼叫 StarDist pretrained model，適合病理 WSI；(c) Napari + scikit-image——pure Python，可載 Cellpose、StarDist、SAM 2 plugin，互動式 QC。pipeline 設計建議：QC 與標註用 GUI（Napari / QuPath），batch inference 用 CLI / Python script，最終 feature extraction 走 scikit-image / CellProfiler 並導 CSV 進 R 做統計。Schmid 2024 (Nat Methods) 的 BioImage Model Zoo 文章詳細介紹這種跨平台流程。

④ JUMP-Cell Painting benchmark（Chandrasekaran 2023 Nat Methods）。JUMP（Joint Undertaking in Morphological Profiling）是 Broad Institute 領銜的 cell painting 大型 benchmark：12 個 cell line × 116,750 chemical perturbations × 6 fluorescence channels（Hoechst, ConA, SYTO14, MitoTracker, phalloidin, WGA），3.5 億 cell images。標準流程：CellProfiler 抽 ~1,000 features per cell → average per perturbation → mean Average Precision (mAP) 評估「同 MoA 化合物是否聚在一起」。是評估 cell painting AI 模型（如 DINO4Cells、Cell Painting Gallery）的 de facto standard。最新挑戰：foundation model（CellSAM + DINOv2 embedding）能否在 JUMP 上 zero-shot 比 CellProfiler hand-crafted features 強？2025 多項初步結果顯示 foundation embedding 在「rare MoA discovery」上有 1.5–2× 提升。

① SAM ViT architecture (Kirillov 2023). SAM has three parts: (a) Image Encoder — MAE-pretrained ViT-H/L/B (632M / 308M / 86M params), 1024×1024 input → 64×64×256 embedding; expensive but runs once per image (cacheable). (b) Prompt Encoder — embeds point / box / mask into 256-d; sparse (point/box) uses positional encoding, dense (mask) uses conv embedding. (c) Mask Decoder — a two-way attention transformer (~4M params) fusing image + prompt embeddings, outputting 3 masks + IoU predictions — lightweight and real-time. This design lets you "encode once, prompt infinitely", which is why SAM works in interactive tools like Napari and ImageJ.

② Cellpose flow-field algorithm (Stringer 2021). Classical instance seg uses anchors / boxes; Cellpose instead predicts a "flow field" — a 2D vector at each pixel pointing toward its cell centre. Ground-truth flow is computed from ground-truth masks via heat diffusion (flowing toward mask centres). At inference: (a) predict flow + cellprob; (b) for each pixel, run Euler integration 200 steps along the flow; (c) endpoints converge to mask centres, forming clusters that naturally yield instance IDs. Advantages: works on arbitrary shapes / topologies (elongated, curved, with holes) — unlike StarDist which is restricted to star-convex. Cellpose-SAM keeps this decoder, only swapping the backbone for SAM ViT.

Schmid 2024 (Nat Methods) on the BioImage Model Zoo describes this cross-platform flow in detail.
